# Core Beliefs

## Agent-First Principles
- The repository must be understandable by a new agent in minutes.
- Written rules beat implicit tribal knowledge.
- Decisions should be captured where future agents will look first.

## Engineering Beliefs
- Small, reversible changes are preferred.
- Mechanical checks are preferred over subjective judgment where possible.
- Repeated workflows should become scripted workflows.

## Documentation Beliefs
- `AGENTS.md` is a map, not a manual.
- `docs/index.md` is the entrypoint for progressive disclosure.
- Avoid duplication; link to a single source of truth.

## Operational Beliefs
- If it matters, encode it in `docs/`, `skills/`, or CI.
- Unknowns should be explicit `TODO`s, not hidden assumptions.
- Post-incident learning should update runbooks and invariants.
